# n8n-nodes-videosailor — Claude Code Guide

## What This Is
Custom n8n community node wrapping the [VideoSailor API](https://api.videosailor.com). Single node, 8 operations, credential-based auth.

## Project Structure
```
n8n-node/
  nodes/VideoSailor/
    VideoSailor.node.ts       — Main node (all operations + execute logic)
    videosailor.svg
  credentials/
    VideoSailorApi.credentials.ts — API key + base URL credential
  dist/                       — Build output (tsc + gulp copies icons)
  gulpfile.js                 — Copies nodes/**/*.{png,svg} to dist/
  package.json                — Package: n8n-nodes-videosailor
  tsconfig.json
  README.md
```

## Operations (single node, operation dropdown)
| Operation | API Flow | Output |
|-----------|----------|--------|
| Download | `POST /api/download` → `GET downloadUrl` | Binary file (`binaryPropertyName` param, default `data`) + JSON metadata |
| Video Info | `POST /api/video` | JSON (title, duration, thumbnails) |
| Trim | `POST /api/trim` | JSON |
| Cut | `POST /api/cut` | JSON (alias for trim) |
| Resize | `POST /api/resize` | JSON |
| Transcribe | `POST /api/transcribe` | JSON (word/segment timestamps) |
| Subtitles | `POST /api/subtitles` | JSON |
| Info | `POST /api/info` | JSON (dimensions, aspect ratio) |

## Key Implementation Details

### Download (binary output)
Two-step process in a single node execution:
1. `POST /api/download` with `{ url }` → returns `{ downloadUrl, filename }`
2. `GET downloadUrl` with `encoding: 'arraybuffer', json: false` → raw Buffer
3. `this.helpers.prepareBinaryData(buffer, filename)` → n8n binary object
4. Output: `{ json: { filename, downloadUrl }, binary: { [binaryPropertyName]: binaryData } }`

### Credential structure
- `apiKey` — sent as `X-API-Key` header (httpHeaderAuth)
- `baseUrl` — default `https://api.videosailor.com`, overridable for local dev

### n8n node type
- Internal name: `videoSailor`
- Published full type in workflows: `n8n-nodes-videosailor.videoSailor`
- Locally-linked type (before publish): `CUSTOM.videoSailor` — n8n emits this prefix for linked custom nodes. Any workflow JSON built against a linked build must be updated once the package is installed from npm.
- Icon: `file:videosailor.svg`
- `usableAsTool: true` (can be used as AI agent tool)

### Helper functions (bottom of file)
- `buildRequest(baseUrl, method, path, body?)` — constructs `IHttpRequestOptions` with JSON headers
- `validateTimeRange(ctx, index, start, end)` — throws if `end <= start`

## Build & Link
```bash
npm run build          # tsc && gulp build:icons
npm link               # register globally
cd ~/.n8n/custom && npm link n8n-nodes-videosailor  # link into n8n
# Restart n8n after linking or rebuilding
```

## Constraints
- **Single Download operation** — do not split into separate "Download" and "Download Media". One node per logical capability keeps the UX tight.
- `@types/node` is installed with `--ignore-scripts` to avoid `isolated-vm` native build errors.
- All auth goes through `this.helpers.httpRequestWithAuthentication` with credential name `'videoSailorApi'`.
- Do not add DB access, long-running jobs, or external I/O beyond the VideoSailor API to this node — it's a thin wrapper by design.
- Do not edit `dist/` by hand; it's regenerated by `npm run build`.

## AI Coding Instructions

### When changing the node
1. Edit `nodes/VideoSailor/VideoSailor.node.ts` (and `credentials/` if auth shape changes).
2. `npm run build` — verify tsc + gulp succeed.
3. Restart the linked n8n instance so the new build is picked up.
4. If any consumer workflow references a parameter, operation name, credential key, or the node `type` you just changed, update those workflows and re-import them.

### Non-negotiables
- Never rename the internal node name (`videoSailor`) or package name (`n8n-nodes-videosailor`) without updating `type` references in every workflow JSON that consumes it.
- Never split the Download operation (see Constraints).
